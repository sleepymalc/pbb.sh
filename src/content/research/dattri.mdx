## Problem

Data attribution—understanding which training samples drive a model's predictions—has become central to improving AI performance and safety. The field has produced a growing family of methods, from influence functions to TracIn to TRAK, yet each is typically released as a standalone implementation with its own interfaces, assumptions, and evaluation protocol. There is no shared library that makes these methods easy to use, compare, or build on. Practitioners who want to apply attribution in an existing pipeline must navigate incompatible codebases; researchers developing new methods must reimplement the same low-level primitives from scratch; and the community has no standardized benchmark against which competing approaches can be fairly evaluated at realistic model scales.

## Key Idea

**dattri** is an open-source PyTorch library that unifies how data attribution methods are written, extended, and benchmarked. Rather than privileging any single method, it provides shared infrastructure that serves three distinct constituencies at once: practitioners who need seamless integration into existing training pipelines, researchers who need reusable building blocks for developing new methods, and the broader community that needs reproducible, apples-to-apples comparisons across the state of the art.

## Approach

**dattri** is organized around three complementary pillars.

The first is a *unified API*. Users define an `AttributionTask` object that captures the loss function and trained checkpoints, then instantiate any supported `Attributor` with a few lines of code. Switching between methods requires no structural changes to the surrounding pipeline — the library is designed for minimal code invasion.

The second is a library of *modular low-level utilities*. Hessian-vector products (HVPs), inverse-HVPs (IHVPs), random projection, and dropout ensemble support are all exposed as standalone, composable components built on `torch.func`. Researchers building new attribution methods can reach for these primitives directly rather than reimplementing them, keeping new work focused on the algorithmic contribution.

The third is a *benchmark suite*. **dattri** ships with pre-trained models and ground-truth attribution annotations across diverse settings — image classification (MNIST, CIFAR), music generation (MAESTRO), and text generation — including generative AI tasks that earlier benchmarks ignored. Evaluation metrics cover leave-one-out (LOO) correlation, linear datamodeling score (LDS), and noisy label detection AUC, providing a multidimensional view of method quality without requiring expensive model retraining.

## Result

**dattri** makes it possible to benchmark all major efficient attribution method families — Influence Function, TracIn, Representer Point Selection, and TRAK — under a single, controlled evaluation framework at large neural-network scales. The resulting comparisons surface concrete, reproducible findings: TRAK's ensembling strategy gives it an edge across most non-linear settings, while traditional LOO metrics prove unreliable in non-convex regimes, motivating the use of LDS as a more faithful signal. Beyond the benchmark itself, the library lowers the barrier to entry for both practitioners deploying attribution in real systems and researchers building the next generation of methods.
